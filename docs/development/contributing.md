# Contributing to ContainerPub

Thank you for your interest in contributing! This guide provides an overview of how to contribute to the project.

## How to Contribute

- **Report bugs**: If you find a bug, please open an issue on GitHub.
- **Suggest features**: Have an idea? Open an issue to discuss it.
- **Submit pull requests**: We welcome code contributions. Please follow the guidelines below.

## Development Setup

1.  **Fork and clone the repository:**

    ```bash
    git clone https://github.com/your-username/ContainerPub.git
    cd ContainerPub
    ```

2.  **Set up the local environment:**

    ```bash
    ./setup-local.sh
    ```

3.  **Run the backend in development mode:**

    ```bash
    make start-backend
    ```

4.  **Make your changes** and test them locally.

## Pull Request Process

1.  Ensure your code follows the project's coding style.
2.  Write tests for your changes.
3.  Update the documentation if necessary.
4.  Open a pull request with a clear description of your changes.
